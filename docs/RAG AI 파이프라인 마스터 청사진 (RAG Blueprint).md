# 📘 04_RAG AI 파이프라인 마스터 청사진 (RAG Blueprint)

본 문서는 쇼핑 리뷰 챗봇 프로젝트를 통해 정립한 RAG(Retrieval-Augmented Generation, 검색 증강 생성) 아키텍처의 표준 워크플로우와 기술 스택을 정의한 단일 진실 공급원입니다. 단순 CRUD를 넘어, 자체 데이터를 기반으로 답변하는 AI 서비스를 구축할 때 AI 에이전트의 기술 지침서로 기능합니다.

## 1. 🚀 RAG 시스템 코어 아키텍처

AI가 헛소리(할루시네이션)를 하지 않고, 우리의 DB를 기반으로 정확히 답변하게 만드는 3단계 코어 파이프라인입니다.

1. **지식 주입 (Indexing)**: 원본 데이터(CSV 등)를 쪼개고(Chunking), 임베딩 모델(예: text-embedding-v2)을 통해 다차원 벡터 좌표로 변환한 뒤 벡터 DB에 저장합니다.
    
2. **유사도 검색 (Retrieval)**: 유저가 질문하면, 질문 역시 벡터로 변환하여 벡터 DB에서 가장 의미가 유사한 상위 K개의 문서를 찾아냅니다.
    
3. **증강 생성 (Generation)**: 찾아낸 문서(근거)와 유저의 질문을 하나로 합쳐 거대한 프롬프트를 만든 뒤, LLM(Gemini Flash 등)에 던져 최종 답변을 생성합니다.
    

## 2. 🛠️ 표준 기술 스택 및 연동 원칙

특정 벤더에 종속되지 않고 유연성을 확보하기 위한 스택 전략입니다.

- **Orchestration**: **LangChain (필수)**
    
    - 절대 AI 제공업체(OpenAI, Pinecone 등)의 Raw SDK를 직접 하드코딩하지 마십시오. 반드시 LangChain의 추상화된 모듈(`langchain-core`, `langchain-community`)을 통해 연결해야 향후 모델이나 DB 교체 시 코드를 한 줄만 수정하여 대응할 수 있습니다.
        
- **LLM (생성용)**: Gemini 2.5 Flash 등 클라우드 API 사용 (웹앱 환경에서 로컬 LLM은 속도 병목을 유발하므로 지양).
    
- **Vector DB**:
    
    - (초기) Pinecone을 활용한 독립적 벡터 저장소 운용.
        
    - (디벨롭) Supabase의 `pgvector` 확장을 활용하여 기존 관계형 DB와 벡터 DB를 단일 인프라로 통합 권장.
        

## 3. 🎨 AI 네이티브 UI/UX 가이드라인 (ZONE C)

AI 챗봇 인터페이스 구축 시 반드시 포함되어야 할 사용자 경험 원칙입니다.

1. **상태 피드백 (Typing Indicator)**: RAG 파이프라인은 시간이 걸립니다. 대기 시간에 반드시 스켈레톤 UI나 타이핑 로딩 애니메이션(pulse)을 제공해야 합니다.
    
2. **근거의 시각화 (Source Card)**: AI가 답변을 생성할 때 참고한 리뷰나 문서를, 답변 하단에 아코디언 형태의 '참고한 리뷰(Source Card)'로 반드시 시각화하여 정보의 신뢰도를 높입니다.
    
3. **Markdown 렌더링**: LLM이 반환하는 답변은 볼드체, 리스트 등이 포함된 마크다운이므로 `react-markdown` 라이브러리를 활용해 깔끔하게 렌더링합니다.
    

## 4. 🧠 실전 디벨롭 및 트러블슈팅

1. **컨텍스트 오염 방지**: 유저의 질문이 기존 데이터와 전혀 관련 없을 경우, AI가 억지로 답변을 지어내지 않도록 시스템 프롬프트에 _"제공된 문서에 답이 없다면 모른다고 답변할 것"_ 이라는 가드레일을 반드시 추가합니다.
    
2. **대화 기억 (Memory)**: 단발성 질문을 넘어 핑퐁 대화를 구축하려면, LangChain의 `BufferMemory`나 DB 기반의 세션 히스토리 저장 로직을 추가하여 이전 맥락을 프롬프트에 지속적으로 포함시켜야 합니다.
    
3. **Edge Runtime 주의**: Next.js의 Edge 환경에서는 일부 Node.js 전용 라이브러리(fs 모듈을 쓰는 CSV 파서 등)가 작동하지 않을 수 있으므로, 서버 액션이나 Route Handler 환경 설정에 주의합니다.
