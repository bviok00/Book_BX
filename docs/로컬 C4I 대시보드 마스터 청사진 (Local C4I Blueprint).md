# 로컬 C4I 대시보드 마스터 청사진 (Local C4I Blueprint)

## 개요
본 청사진은 클라우드 종속성을 철저히 배제하고, 로컬 파일 시스템, SQLite 데이터베이스, 로컬 Ollama 모델을 유기적으로 연동하여 **"완전 자립형 C4I 하이브리드 대시보드"**를 성공적으로 구축한 마스터 개발 워크플로우와 설계 철학을 문서화한 것입니다. 차기 로컬 퍼스트 AI 프로젝트에서 시행착오 없이 즉각적으로 적용할 수 있는 표준 개발 바이블로 활용됩니다.

---

## 1. 5-ZONE 로컬 하이브리드 아키텍처 설계
시스템의 복잡성을 통제하고 에러 확산을 원천 차단하기 위해 화면 레이어와 비즈니스 로직, 데이터 Plumber를 완벽히 격리하는 **5-ZONE 아키텍처**를 규격화합니다.

```
📂 supabase/                 ← ZONE A: 지하실 (가벼운 클라우드 Auth & 세션 관리)
📂 src/app/                  ← ZONE B: 엔진실 (서버 컴포넌트, Route Handler, Server Actions)
📂 src/components/           ← ZONE C: 쇼룸 (다크 커맨드 센터 UI, Recharts, Client Components)
📂 src/lib/                  ← ZONE D: 배관실 (better-sqlite3 직통 커넥션, fs/promises 파일 입출력)
📂 src/types/                ← ZONE E: 명부실 (도메인 모델 및 TypeScript 공통 타입 선언)
```

### 아키텍처 계층별 책임 및 설계 원칙
*   **ZONE A**: Supabase Auth 및 가벼운 채팅 메타데이터 관리 전용으로 사용하며 무료 티어 용량 한도(500MB)를 영구히 유지합니다.
*   **ZONE B**: Next.js 16 / React 19 서버 인프라를 백엔드 컨트롤 센터로 활용하고 비가역적 파이썬 실행(`child_process`) 및 LLM Fetching을 캡슐화합니다.
*   **ZONE C**: 브라우저 default 폰트를 완전히 밀어내고 고딕 및 Monospace 혼합 글꼴을 적용하며 UI 컴포넌트가 150줄을 넘지 않도록 철저히 모듈화합니다.
*   **ZONE D**: ORM 레이어를 제거하고 raw SQL 기반 `better-sqlite3` 싱글톤 패턴과 비동기 `fs/promises`를 연계해 디스크에 직접 읽기/쓰기를 수행합니다.
*   **ZONE E**: API 통신 규격 및 DB 레코드 스키마 타입을 하나로 묶어 Type-safe 컴파일 환경을 강제합니다.

## 2. 로컬 데이터 주권 수호 및 배관 설계
로컬 PC의 파일 시스템을 신뢰성 있는 저장소(Single Source of Truth)로 사용하고, 대용량 DB는 Local에, 가벼운 세션만 Cloud에 두는 하이브리드 형태를 구축합니다.

*   **better-sqlite3 직통 접근**: Vercel 등 외부 서버 리소스 없이 로컬 PC 자체에서 `npm run start`를 구동하므로 무거운 API 요청 없이 SQLite DB에 1ms 내외로 즉시 쿼리를 주입하여 처리 속도를 최적화합니다.
*   **FS 기반 마크다운 위키 저장소**: 투자 심의 리포트 전문을 무겁게 DB에 텍스트로 보관하지 않고 `brain_BX/wiki/ai_report/stock_report/` 경로 아래 개별 마크다운 파일로 직접 쓰고 조회합니다. 파일 자체가 유일한 원천 데이터가 됩니다.
*   **암호화 연동**: 로컬 개발망 외부로의 API 키 유출을 방지하기 위해 Supabase에 사용자별 API 키를 대칭키 방식으로 암호화하여 저장하거나 로컬 PC의 `.env.local`로 관리합니다.

## 3. 실시간 로그/응답 SSE 스트리밍 설계 (ReadableStream)
로컬 Ollama LLM 추론 시간(수십 초) 및 백엔드 파이썬 배치 스크립트 실행 로그(수 분)를 대기 시간 없이 실시간으로 중계하는 고성능 스트리밍 통신 배관을 설계합니다.

### 아키텍처 흐름도
```
[Next.js Server Actions / API] ── child_process.spawn ──> [Python Pipeline Engine]
              │                                                     │
              ▼ (ReadableStream 버퍼 가로채기)                        ▼ (stdout/stderr 실시간 출력)
[Server-Sent Events (SSE) 라우터] ───────────────────────────> [브라우저 EventSource UI]
```

### 핵심 구현 방법
1.  **Node.js child_process.spawn 활용**: 백그라운드 프로세스를 비동기 기동하고 `proc.stdout` 버퍼 스트림을 가로챕니다.
2.  **ReadableStream 포장**: Next.js Route Handler 내에서 버퍼 데이터 청크를 `data: ${text}\n\n` 형태로 감싸 웹 표준 스트림 규격으로 래핑합니다.
3.  **SSE 통신 규격 준수**: Response 헤더를 `Content-Type: text/event-stream`으로 반환하여 프론트엔드가 실시간 터미널 뷰어를 구성할 수 있게 배관합니다.

## 4. 13인 AI 에이전트 위원회 구축 & 프롬프트 설계
본 프로젝트의 가장 강력한 코어 시스템은 13인의 독자적인 전문 지능 에이전트가 단일 심의를 거쳐 다각도 투자 보고서를 뽑아내는 **오케스트레이션 엔진**입니다.

### 13인 요원단 보직 설계 및 프롬프트 명세
1.  `01_HQ_Commander` (지휘참모): 전체 분석 범위 획정 및 분석 세부 방향성 지휘.
2.  `02_HQ_Mentor` (전략참모): 경험적 통찰 기반 실전 포트폴리오 가이드라인 제공.
3.  `03_HQ_Researcher` (연구참모): SQLite DB 및 인텔리전스 파일 스캔 및 팩트 서칭.
4.  `04_HQ_Developer` (분석참모): 데이터 분석을 돕는 코드 도구 및 마크다운 템플릿 개발.
5.  `05_Invest_Chairman` (심의의장): 요원 간 상충하는 의견 조율 및 최종 합의안 최종 의결.
6.  `06_Invest_Timing` (타이밍참모): RSI, MACD, 이동평균선 이격도, 밴드 활용 단기 타점 진단.
7.  `07_Invest_Trend` (추세참모): 200일 장기 이평선 기준의 추세 평가 및 분할 매수 밴드 설계.
8.  `08_Invest_Risk` (리스크참모): ATR 변동성 경고 및 최악의 하락 손절 마지노선 구축.
9.  `09_Invest_Sector` (섹터참모): S&P500 대비 상대강도(RS) 연산 및 주도 섹터 대장주 스캔.
10. `10_Invest_Buffett` (가치참모): ROE, 영업이익률 15% 잣대 및 경제적 해자 판정.
11. `11_Invest_Lynch` (성장참모): 분기 매출/EPS YoY 성장률 및 PEG 지표 적정성 판정.
12. `12_Invest_Munger` (반전참모): 찰리 멍거의 역발상(Invert)에 기초한 투자의 치명적 함정 진단.
13. `13_Idea_Partner` (아이디어참모): 사령관의 수동 통찰 메모 및 아이디어 융합 연계 브레インスト리밍.

## 5. 고급 전술적 인터랙션 결속 패턴 (Phase 5+)
차세대 대시보드 구축 시 유저가 감동하는 UX를 생성하는 핵심 전술적 UI/UX 배관 패턴입니다.

### 1️⃣ 인터랙티브 액션 링커 (Analytics ➔ Chat 배관)
*   **패턴 설명**: 특정 정량 시각화 화면에서 버튼 클릭 한 번으로 관련 RAG 컨텍스트를 담은 채 챗봇 대화 화면으로 라우팅되는 지능형 연계 기술입니다.
*   **구현 방법**: URL Search Parameters로 질의 내용을 인코딩하여 화면을 전환하고, 대상 컴포넌트의 `useEffect` 훅에서 입력 폼 바인딩 및 포커싱을 강제합니다. 그 직후 `window.history.replaceState`를 호출해 URL 쿼리를 제거함으로써 사용자가 새로고침하더라도 동일 질문이 중복 기동되는 부작용을 깔끔하게 차단합니다.

### 2️⃣ 클라이언트 실시간 사전(Lexicon) 감성 판독기
*   **패턴 설명**: 백엔드 API 호출로 인한 대기 시간을 없애기 위해 금융 뉴스 데이터를 렌더링하는 시점에 실시간으로 감성을 즉각 분류하여 Notion 스타일 태그로 띄웁니다.
*   **구현 방법**: 긍정(수익, 급등, 개선 등) 및 부정(하락, 지연, 실패 등) 키워드 맵을 구축하고 제목의 텍스트가 수신되는 즉시 클라이언트 단에서 매칭 연산을 수행해 🟢 호재, 🔴 악재, 🟡 중립 유리 질감 뱃지를 0.01초 내에 자동 부착합니다.

### 3️⃣ SSE 기반 실시간 요원 작전 상황판 (Stateful Tracker)
*   **패턴 설명**: 여러 에이전트가 작동하는 파이프라인의 진행 상황을 텍스트 로그가 아닌 동적인 램프 점멸 화면으로 시각화합니다.
*   **구현 방법**: SSE 백엔드 출력 로그에서 특정 에이전트의 시그니처 텍스트(예: `[Researcher] DB 스캔 완료`)를 정규식으로 캐치하고 이를 React의 13개 요원 State 상태로 업데이트합니다. 상태가 바뀐 참모 카드 옆에는 `@keyframes pulse-yellow` 애니메이션을 연동한 노란색 램프가 깜빡이게 처리하여 생동감을 선사합니다.

### 4️⃣ OS-Native 스케줄 연계 및 UI 정보 노출
*   **패턴 설명**: 상시 구동 스케줄러가 서버의 메모리를 잠식하는 현상을 완벽히 차단하기 위해 OS 스케줄러에 실행 권한을 양도하고, UI에는 각 작업의 스케줄링 메타데이터를 정밀히 매칭시킵니다.
*   **구현 방법**: Windows 작업 스케줄러에 `run_pipeline.py --task <task>`를 연쇄적으로 설정해 백그라운드 구동을 자동화합니다. 파이프라인 관리 화면에서는 각 작업의 스케줄링 현황(`⏱️ KIS 연결 전 수동`, `⏱️ 매일 오전 6시 자동`, `⏱️ Refine 실행 시 후속 자동 연계`)을 눈에 잘 들어오는 미니 배지로 직관화합니다.

## 6. 마스터 개발 워크플로우 7단계
미래의 신규 로컬 퍼스트 AI 대시보드 프로젝트 개시 시 본 가이드의 워크플로우를 그대로 주입하여 구현합니다.

*   **1단계 [개발 환경 구축]**: Node.js 및 Python 가상환경을 구축하고 `package.json`의 빌드/기동 명령어를 세팅합니다.
*   **2단계 [디자인 시스템 정의]**: `globals.css` 파일에 암흑 사령부 컨셉의 HSL CSS 변수를 완벽하게 정렬하고, 각 레이블의 끝에 Monospace 및 각진 Orbitron 폰트를 인프라스트럭처 수준에서 강제합니다.
*   **3단계 [데이터 plumbings 개척]**: `better-sqlite3` 싱글톤 연결 코드를 만들고 `fs/promises`를 사용하는 위키/요원 리포터 입출력 모듈을 설계합니다.
*   **4단계 [서버 브릿지 연동]**: Python 스크립트 실행 버퍼를 받아서 브라우저에 쏴주는 SSE Route Handler 배관을 뚫고 실시간 stdout 중계를 완성합니다.
*   **5단계 [C4I UI 뷰포트 빌딩]**: `/` 관제탑, `/chat` 챗봇, `/committee` 요원 상황판, `/pipeline` 대시보드를 단계적으로 클라이언트 컴포넌트로 분할 조립합니다.
*   **6단계 [전술적 인터랙션 결속]**: 액션 링커, 실시간 감성 뱃지 판독기, SSE Stateful 요원 램프 점멸기, OS 스케줄러 메타데이터 연동을 순차적으로 튜닝합니다.
*   **7단계 [프로덕션 빌드 및 검증]**: PWA 구동을 위한 `manifest` 무결성을 확인하고 `npm run build`를 기동하여 정적 컴파일 단계의 컴파일 에러와 TypeScript 타입 누수를 최종 청소합니다.
