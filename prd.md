# 🎖️ [PRD v4.0] Intellect_BX — 시네마틱 하이브리드 지식 관제탑

> **설계 철학:** "연구실(Notion Light)과 영화관(Watcha Dark)을 오가는 Two-Face 하이브리드 UI + AI 인텔리전스 레이어"

---

## 1. 프로젝트 개요 (Overview)

|항목|내용|
|---|---|
|**목적**|알라딘 API로 도서 데이터를 수집하고, 계층형 독서 노트를 기록하며, 이를 옵시디언 형태의 인터랙티브 지식 그래프로 시각화하는 클라우드 기반 Second Brain 구축|
|**배포**|Vercel (웹앱) + PWA (모바일/태블릿 독립 구동)|
|**인증**|Supabase Auth — Google OAuth 단일 진입점|
|**보안**|전 테이블 RLS(Row Level Security) 강제 적용|
|**디자인 철학**|Light = Notion 오프화이트 연구실 / Dark = Watcha 시네마틱 영화관 — 150ms 트랜지션으로 전환|

---

## 2. 기술 스택 (Tech Stack)

|레이어|기술|용도|
|---|---|---|
|**Frontend**|Next.js 15 (App Router), React 19, Tailwind CSS|전체 UI 프레임|
|**테마**|`next-themes`|Light/Dark 전역 스위칭|
|**Backend/DB**|Supabase (PostgreSQL + Auth + Storage)|DB, 인증, 이미지 스토리지|
|**네트워크 시각화**|`react-force-graph` or `d3-force`|지식 성단 마인드맵|
|**차트**|`recharts`|레이더 차트, 히트맵 연산|
|**이미지 처리**|`color-thief`|책 표지 dominant color 추출|
|**공유 카드**|`html-to-canvas`|메모 인스타그래머블 카드 생성|
|**외부 API**|알라딘 Open API|도서 검색, 표지, 목차, 메타데이터|
|**AI**|Anthropic Claude API (claude-sonnet)|태그 추천, Summary 초안, 연결 도서 추천|
|**익스포트**|`jszip` + `file-saver`|Obsidian 마크다운 번들 내보내기|

> **알라딘 API CORS 방어:** 브라우저 직접 호출 불가. Next.js Route Handler(`/api/alaadin`)를 프록시로 반드시 구축.

## 4. 4-ZONE 대시보드 상세 기능 명세

### 📡 ZONE 1: 헤더 — 지적 전투력 검증

#### 4.1.1 지식 잔디 히트맵

- `reading_sessions` 테이블 기준으로 독서 활동일 추적 (단순 `updated_at` 오염 방지)
- GitHub 잔디 스타일 1년치 캘린더 렌더링
- 활성일 호버 시 해당 날짜 독서 시간 + 메모 개수 툴팁 표시

#### 4.1.2 지식 레이더 차트 (Cognitive Radar)

- 완독 도서의 카테고리(철학, 기술, 경제 등) 실시간 집계
- `recharts` RadarChart — 6각형 시각화
- 클릭 시 해당 카테고리 도서만 중앙 패널에 필터링

#### 4.1.3 연간 독서 목표 게이지

- 연초 목표 권수 설정 (예: `2026년 목표 50권`)
- 헤더 우측에 `23/50권 — 46%` 실시간 진행률 바
- 완독 시 컨페티 애니메이션 보상 트리거

#### 4.1.4 오늘의 지식 회상 (Spaced Repetition)

- 과거 마이크로 메모 중 랜덤 1개를 매일 헤더에 카드로 노출
- 알고리즘: 최근 30일 내 조회되지 않은 메모 우선 선택
- `[💡 오늘의 인사이트]` 섹션 — 클릭 시 원본 책 디테일 뷰로 이동

#### 4.1.5 외부 공유 토큰

- `[🔗 내 서재 공유]` 버튼 → 고유 해시 토큰 포함 `Public Read-only URL` 생성
- 공유 뷰: 서재 + 공개 설정된 독서 노트만 열람 가능 (편집 불가)

---

### 🔭 ZONE 2: 좌측 패널 — 폴더 트리 & 상태 관리

#### 4.2.1 물리적 폴더 트리 (Context)

- 도서가 소속되는 단일 프로젝트 단위 관리 (1:N 관계 엄수)
- 예: `[2026 거시경제 연구]`, `[AI 아키텍처 고도화]`, `[철학 탐사대]`
- Notion 스타일 인라인 이름 수정, 드래그 재정렬 지원

#### 4.2.2 독서 상태 트래킹 (4-State Machine)

```
WANT_TO_READ → READING → COMPLETED
                       ↘ DROPPED
```

- 수동 퍼센트(%) 입력 완전 배제 — UX 마찰 제거
- 상태 변경 시 `reading_sessions`에 이벤트 로그 자동 기록

#### 4.2.3 타겟 록온 (Wishlist Queue)

- 알라딘 검색 → `WANT_TO_READ` 상태로 즉시 등록
- 대기열 우선순위 드래그 재정렬 지원

---

### 🧠 ZONE 3: 중앙 메인 패널 — 시네마틱 지식 용광로 (핵심)

#### 4.3.1 Three-Track 뷰 스위치

|뷰|아이콘|스타일|용도|
|---|---|---|---|
|**그리드 뷰**|`☷`|Watcha 포스터 히어로 배너 + 카루셀|탐색/수집|
|**리스트 뷰**|`☰`|고밀도 텍스트 테이블|분석/관리|
|**책등 뷰**|`📚`|실제 책장 스파인(책등) 나열|컬렉션 보상감|

**그리드 뷰 상세:**

- 상단: 현재 `READING` 상태 책을 와이드 히어로 배너로 배치
- 하단: 폴더/카테고리별 가로 스와이프 카루셀
- 책 클릭 → 표지 이미지 배경 블러(`backdrop-filter: blur`) + 웅장한 디테일 뷰 오버레이
- 책 표지 로드 시 `color-thief`로 dominant color 추출 → 디테일 뷰 accent/배경에 적용

**책등 뷰 상세:**

- 완독 권수가 쌓일수록 책장이 꽉 차는 시각적 보상 구조
- 책등 호버 시 제목/저자 툴팁, 클릭 시 디테일 뷰 진입

#### 4.3.2 계층형 독서 노트 아키텍처

```
Book (1권)
├── Summary Note (단일 총평, 1개) — 완독 후 최종 인사이트
└── Chapters (목차 구조)
    ├── Chapter 1
    │   ├── Granular Note (마이크로 메모, N개)
    │   └── Granular Note ...
    └── Chapter N
        └── Granular Note ...
```

**목차 폴백 엔진:**

- 알라딘 API TOC 존재 시: JSONB 구조 자동 파싱 후 챕터 렌더링
- TOC 부재 시(에세이/만화 등): `[+ 챕터 직접 추가]` 버튼 활성화 — 사용자 수동 구조 개설

**마이크로 메모 UX:**

- 왓챠 코멘트 피드 스타일 타이포그래피
- 메모 저장 시 AI 태그 추천 트리거 (3개 추천 → 1클릭 확정)
- 페이지 참조 수동 입력 (`p.142` 형식)
- `[📤 공유 카드]` 버튼: `html-to-canvas`로 책 표지 + 메모 인스타그래머블 이미지 생성

#### 4.3.3 포커스 모드 (독서 세션)

- `READING` 상태 책에서 `[🎯 포커스 모드]` 진입
- 전체 UI 제거 → 책 표지 + 세션 타이머 + 메모창만 풀스크린 렌더링
- 세션 종료 시 독서 시간 `reading_sessions` 자동 기록

---

### 🔗 ZONE 4: 우측 패널 — 지식 네트워크 터미널

#### 4.4.1 옵시디언 미니 지식 성단

- `react-force-graph` 기반 2D/3D 인터랙티브 그래프
- 노드 타입: `Book` (원형, 표지 썸네일) / `Tag` (육각형, 개념 키워드)
- 간선: `book_tags`(책-태그), `note_tags`(메모-태그) 기반 연결선 렌더링
- 그래프 자전 애니메이션 (기본 천천히 회전)
- 태그 노드 클릭 → 중앙 패널에 해당 태그 보유 도서 칼같이 필터링

#### 4.4.2 개념 태그 시스템

- 폴더 트리(ZONE 2)와 완전 분리 — 폴더 = 컨텍스트, 태그 = 지식 원소
- 책 레벨 + 메모 레벨 모두 N:M 다중 태그 부여 가능
- 예: `#행동경제학`, `#도파민`, `#리스크관리`, `#1차사고vs2차사고`

#### 4.4.3 Obsidian 마크다운 익스포트

- `[📥 Obsidian 내보내기]` 버튼
- 전체 서재를 책별 `.md` 파일로 변환 후 `jszip`으로 번들링 → 다운로드
- 파일 구조:

```
intellect_bx_export/
├── 책제목_저자.md      ← Summary + 메모 전체 포함
├── 책제목2_저자.md
└── ...
```

- 각 `.md` 파일에 Obsidian 호환 `[[태그]]` 링크 자동 삽입

## 5. AI 인텔리전스 레이어 (Claude API 연동)

> 모든 AI 호출은 Next.js Route Handler(`/api/ai/*`)를 통해 서버사이드 처리. API Key 클라이언트 노출 금지.

|기능|트리거|입력 컨텍스트|출력|
|---|---|---|---|
|**메모 자동 태깅**|마이크로 메모 저장 시|메모 텍스트 + 책 제목/장르|관련 태그 3개 추천 (확정/기각 선택)|
|**Summary 초안 생성**|`[✨ AI 초안]` 버튼 클릭|해당 책 마이크로 메모 전체|총평 초안 텍스트 생성 (사용자가 수정 후 저장)|
|**연결 도서 추천**|그래프 태그 노드 호버|해당 태그 + 연결된 책 목록|"이 개념을 공유하는 책" 3권 → 알라딘 검색 연동|

## 7. 비기능 요구사항 (NFR)

| 항목             | 기준                                        |
| -------------- | ----------------------------------------- |
| **초기 로드**      | LCP ≤ 2.5s (Vercel Edge + Next.js ISR 활용) |
| **그래프 렌더링**    | 노드 200개 이하에서 60fps 유지                     |
| **테마 전환**      | 150ms 이내 전체 반전 완료                         |
| **알라딘 API 응답** | 서버사이드 캐싱 (책 메타데이터 24시간 Supabase 캐시)       |
| **AI 응답**      | 태그 추천 ≤ 3s, Summary 초안 ≤ 10s (스트리밍 권장)    |
| **보안**         | 전 테이블 RLS 적용, AI API Key 서버사이드 격리         |
