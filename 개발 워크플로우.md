# 📘 AI-Native TDD Master Blueprint (One-Turn Execution)

- One-Turn TDD (초압축 사이클): AI는 텍스트를 출력하기 전 내부 연산으로 Red(실패) 상태를 이미 시뮬레이션할 수 있으므로, 사용자의 승인을 기다리며 대화를 끊지 마라. 단 한 번의 답변(Turn) 내에서 [테스트 코드 작성]과 [통과하는 구현 코드 작성]을 연속으로 모두 출력하여 즉시 Green 상태를 만들어내야 한다.
    
- 테스트 타겟팅 (No E2E): 브라우저의 비동기 렌더링을 직접 제어해야 하는 E2E 테스트(Playwright 등)는 AI의 환각(Hallucination) 루프를 유발하므로 이 단계에서 절대 작성하지 마라.

### 패턴 A: `next.config.ts` — 네이티브 모듈 및 로컬 호스팅 세팅

`better-sqlite3` 같은 Node.js 네이티브 바이너리(.node)를 쓸 때, webpack이 번들링하면 깨집니다. 반드시 `serverExternalPackages`로 제외해야 합니다.

```typescript
// next.config.ts — 로컬 자가 호스팅 + 네이티브 모듈 제외 패턴
const nextConfig: NextConfig = {
  serverExternalPackages: ["better-sqlite3"],  // 네이티브 .node 파일 번들 제외
  images: { unoptimized: true },               // CDN 불필요 시 이미지 최적화 비활성화
  async headers() {
    return [{ source: "/(.*)", headers: [
      { key: "X-Frame-Options", value: "SAMEORIGIN" },
      { key: "X-Content-Type-Options", value: "nosniff" },
    ]}];
  },
};
```

**활용 예시**: 독서 앱에서 SQLite로 읽은 책 목록을 관리하거나, 영화 리뷰 앱에서 로컬 DB에 리뷰를 저장할 때 동일한 세팅을 적용합니다.

### 패턴 B: Server Action 표준 응답 + `revalidatePath` 캐시 무효화

Server Action은 절대 `throw new Error()`로 에러를 터뜨리지 않고, 무조건 `{ success, message }` 규격 객체를 반환합니다. DB 변경 후에는 반드시 `revalidatePath`를 호출하여 UI 자동 갱신을 트리거합니다.

```typescript
'use server';
import { revalidatePath } from 'next/cache';

export async function addBookReview(data: FormData): Promise<{ success: boolean; message: string }> {
  try {
    // DB INSERT 로직...
    revalidatePath('/reviews');     // 리뷰 목록 캐시 무효화
    revalidatePath('/dashboard');   // 대시보드 통계도 갱신
    return { success: true, message: '리뷰가 등록되었습니다.' };
  } catch (error: any) {
    return { success: false, message: `등록 실패: ${error.message}` };
  }
}
```

**절대 금지**: 클라이언트에서 `window.location.reload()`나 `router.refresh()`를 남발하지 않습니다. 데이터 갱신 트리거는 오직 서버 액션 내부의 `revalidatePath`가 통제합니다.

### 패턴 C: SSE 실시간 스트리밍 (ReadableStream) — AI 챗봇 / 장시간 작업 로그

LLM 응답이나 백그라운드 작업의 실시간 진행 상황을 프론트엔드에 끊김 없이 중계하는 핵심 배관입니다.

```
[Route Handler] ── ReadableStream ──> [SSE: text/event-stream] ──> [브라우저 EventSource]
```

**구현 핵심 3단계**:
1. **소스 연결**: `fetch(ollama_url, { body: ... })` 또는 `child_process.spawn(python_script)` 로 데이터 소스를 연결합니다.
2. **스트림 래핑**: `new ReadableStream({ start(controller) { ... } })` 로 청크를 `data: ${text}\n\n` SSE 포맷으로 감쌉니다.
3. **응답 반환**: `new Response(stream, { headers: { 'Content-Type': 'text/event-stream' } })` 으로 브라우저에 전달합니다.

**활용 예시**: 언어학습 앱에서 AI 튜터가 문법을 설명할 때 실시간 스트리밍으로 답변을 출력하거나, 영화 리뷰 앱에서 AI가 영화 분석 리포트를 생성할 때 진행 로그를 실시간 표시합니다.

### 패턴 D: 로컬 파일시스템(fs/promises) — 마크다운 기반 콘텐츠 관리

DB에 저장하기 무겁거나 파일 자체가 원천 데이터인 경우(마크다운 글, AI 리포트, 프롬프트 템플릿 등), `fs/promises`로 직접 읽고 씁니다.

```typescript
// src/lib/fs/content.ts — 범용 마크다운 파일 관리 헬퍼
import { readdir, readFile, writeFile, stat } from 'fs/promises';
import path from 'path';

const CONTENT_DIR = process.env.CONTENT_DIR ?? 'c:/coding/my_project/content';

export async function listMarkdownFiles(): Promise<string[]> {
  const files = await readdir(CONTENT_DIR);
  return files.filter(f => f.endsWith('.md')).sort();
}

export async function readMarkdown(filename: string): Promise<string> {
  const safeName = path.basename(filename);          // 경로 traversal 방지
  return readFile(path.join(CONTENT_DIR, safeName), 'utf-8');
}

export async function writeMarkdown(filename: string, content: string): Promise<void> {
  const safeName = path.basename(filename);
  await writeFile(path.join(CONTENT_DIR, safeName), content, 'utf-8');
}
```

**보안 필수**: `path.basename()`으로 파일명을 정제하여 `../../etc/passwd` 같은 경로 탈출 공격을 원천 차단합니다.

**활용 예시**: 독서 앱에서 읽은 책의 독서 노트를 마크다운으로 저장/조회하거나, 언어학습 앱에서 AI가 생성한 문법 해설 시트를 파일로 아카이브합니다.

한 화면의 분석 결과를 다른 화면(챗봇, 상세페이지 등)으로 1클릭 라우팅하면서 컨텍스트를 자동 주입하는 UX 패턴입니다.

**구현 3단계**:
1. 출발 페이지에서 `router.push('/chat?input=NVDA 분석해줘')` 로 URL에 컨텍스트를 인코딩합니다.
2. 도착 페이지의 Client Component에서 `useSearchParams`로 쿼리를 파싱하고 `useEffect`로 입력 필드에 자동 바인딩 및 포커스를 부여합니다.
3. 처리 완료 직후 `window.history.replaceState({}, '', '/chat')` 으로 URL을 소거하여 새로고침 시 중복 트리거를 방지합니다.

**활용 예시**: 독서 앱에서 독서 통계 화면의 특정 책을 클릭하면 → AI 챗봇으로 넘어가며 "이 책의 핵심 메시지를 요약해줘" 프롬프트가 자동 입력됩니다.

### 패턴 H: 환경변수(.env.local) 경로 중앙 집중 관리

모든 외부 경로와 API 키를 `.env.local` 한 곳에서 관리하고, 코드에서는 `process.env.XXX ?? '기본값'` 패턴으로 참조합니다.

```bash
# .env.local — 경로 중앙 집중 관리 템플릿
DB_PATH=c:/coding/my_project/data/app.db
CONTENT_DIR=c:/coding/my_project/content
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_DEFAULT_MODEL=gemma4:e4b
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

**핵심**: 경로를 하드코딩하면 PC가 바뀌거나 프로젝트를 복사했을 때 전부 깨집니다. `.env.local` 한 파일만 수정하면 전체 시스템이 새 환경에 대응하도록 설계합니다.

### 4-1. 프로젝트 타입별 스택 선택 가이드

- [ ] `src/types/index.ts`에 도메인 타입 정의

### 4-3. 도메인 치환 매핑 (1:N 패턴 범용 적용)

기존 교본의 1:N 관계(운동:세트, 위원회:요원, 파이프라인:작업)를 새 도메인에 그대로 매핑합니다.

| 기존 도메인 (운동 앱) | 독서 앱 | 영화 리뷰 앱 | 언어학습 앱 |
|---|---|---|---|
| 운동 (부모) | 책 (Book) | 영화 (Movie) | 학습 세션 (Session) |
| 세트 (자식) | 독서 노트 (Note) | 리뷰 (Review) | 연습 기록 (Exercise) |
| 루틴 (그룹) | 독서 리스트 (List) | 워치리스트 (Watchlist) | 커리큘럼 (Curriculum) |
| 칼로리 (지표) | 페이지 수 (Pages) | 별점 (Rating) | 정답률 (Accuracy) |

### 4-4. 빌드 최종 검증 순서

프로젝트 완성 후, "완료"라고 말하기 전에 반드시 아래 순서로 무결성을 검증합니다.

1. `npx tsc --noEmit` — TypeScript 컴파일 에러 0건 확인
2. `npm run test` — 단위/통합 테스트 전체 통과 확인
3. `npm run build` — 프로덕션 빌드 성공 확인 (정적 분석 + 번들 최적화)
4. `npm run start` — 실제 구동 후 브라우저에서 핵심 기능 수동 검증
